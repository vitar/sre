# ARTIFACTS

## ARTIFACT - Assumption Excavator - Critical Assumption Set (Multi-Domain)

**Source context:** This assumption set is extracted from “Tri-Contour Dynamics of Limited Resource Allocation” theory (TC).

### A) Applicability & Scope Assumptions (when the model is even “about” the system)

1.  **Assumes that the target system has limited resources** (i.e., resources are meaningfully scarce at the level of analysis).
2.  **Assumes that competing demands exist** such that allocating to one demand reduces what remains for others (trade-offs are real, not illusory).
3.  **Assumes the system exhibits non-static behavior over time** (allocation and/or conditions change; time matters).
4.  **Assumes the model does not apply** when resources are effectively unlimited, demands do not compete, or the system is static.
5.  **Assumes that “allocation” is a coherent central observable** (the system’s behavior can be described as resources going “somewhere”).

### B) Core Ontology Assumptions (what exists in the model)

6.  **Assumes three analytically distinct functional contours exist**: TC names them Survival, Reproduction, Evolution.
7.  **Assumes each contour is definable by “primary functional intent”**, even if activities have multi-contour effects.
8.  **Assumes “resource” can be any limited input** (physical, temporal, energetic, informational, economic, social, institutional, symbolic).
9.  **Assumes “scarcity” is the driver**: without scarcity, contour competition and trade-offs vanish and the model loses explanatory value.
10. **Assumes time has a privileged role** as the medium through which allocation is enacted, and sustained allocation patterns over time create cumulative effects.
11. **Assumes an “environment” exists** as external conditions shaping resource availability, constraints, pressures, and viability—and that the environment can change.
12. **Assumes “mediators” exist as cross-cutting factors** that shape allocation/signaling/constraint/feedback across contours without being a fourth contour.
13. **Assumes mediators do not compete for the same resource pool as contours**; instead they shape how the pool is divided.

### C) Structural Mechanism Assumptions (how allocation is enacted)

14. **Assumes the model requires five structural elements** to explain how allocation works: Code (substrate), Boundary & Gate (structure), Signal & Receiver (dynamics).
15. **Assumes “Code” is the operating logic of the system**, conditioning boundary, gate behavior, signal legibility, and receiver parsing.
16. **Assumes Code is distinct from “Narrative”** and that narrative can change without code changing (Code–Narrative divergence is possible).
17. **Assumes “Boundary” defines inside vs outside** (not necessarily physical; can be organizational/contractual/informational/identity-based).
18. **Assumes boundary persistence has an allocation cost** (maintenance requires continuous expenditure).
19. **Assumes “Gate” controls flows across boundary** (resources, signals, actors, information) and can be selectively permeable, configured by Code.
20. **Assumes “Signal” transmits state-change information** about contour state (distortion, pressure, opportunity, need), but is not inherently meaningful without Code context.
21. **Assumes “Receiver” parses signals** with three properties: Threshold, Legibility Range, and Contour Orientation.
22. **Assumes failures to respond to a signal have multiple structural causes** (Gate blocked vs signal below threshold vs illegible to receiver), and these are diagnostically distinct.

### D) Contour Competition Assumptions (trade-off geometry)

23. **Assumes the three contours compete for the same limited resource pool**, creating permanent structural tensions (Survival↔Evolution, Survival↔Reproduction, Reproduction↔Evolution).
24. **Assumes these tensions arise from scarcity, not necessarily actor conflict**, and persist even with aligned actors.
25. **Assumes the model describes consequences of tension resolution patterns** without prescribing “correct” resolutions.

### E) Dynamic/Causal Assumptions (what tends to happen under pressure)

26. **Assumes “Displacement” occurs** when resource availability declines relative to demand: one contour’s allocation is reduced to serve another.
27. **Assumes displacement order is Code-dependent (context-dependent), not universal**, though the text notes a common human-systems pattern (Evolution first, then Reproduction, then Survival).
28. **Assumes displacement can be gradual yet appear sudden** when masked by compensation, and that the suddenness is an observability artifact.
29. **Assumes “Compensation” exists**: systems can maintain observable viability under distortion by spending finite buffers outside the primary allocation logic.
30. **Assumes compensation has a required structure** (trigger, buffer, signal) and can be adaptive (intentional/bounded) or structural (unintentional/unbounded).
31. **Assumes “Contour Saturation” is a boundary condition** where one contour consumes total allocation space; other contours’ demands persist while allocation becomes structurally impossible.
32. **Assumes saturation implies “allocation logic ceases to function”** (no degrees of freedom remain), and the exit is “system reconstitution,” not reallocation.
33. **Assumes “Feedback” exists and matters**: allocation outcomes feed back to influence future allocation (reinforcing vs balancing).
34. **Assumes feedback efficacy depends on element infrastructure** (signals must cross gates, match legibility, exceed threshold, be parsed within Code).
35. **Assumes “Endogenous Phase Transition” exists**: discontinuous allocation shifts can be triggered by internal accumulation thresholds, even absent external shocks.
36. **Assumes mediators actively modulate dynamics** (amplify, block, redirect mechanisms), and mediators can conflict (mediator tension).
37. **Assumes “Mediator Reciprocity”**: sustained allocation patterns can reconfigure mediators over time via shifts in selectivity, resolution, and temporal behavior.

### F) Metabolic (Time-Accumulation) Assumptions (what builds up)

38. **Assumes sustained contour allocation over time produces accumulation** that connects mechanisms to trajectories.
39. **Assumes three accumulation types**: Debt, Potential, Overhead.
40. **Assumes “Debt” is produced by sustained under-allocation** to a contour and manifests as concrete capability loss (not merely metaphor).
41. **Assumes “Evolution debt” has an accelerating (approximately exponential) curve with perception delay**, driven by compounding environmental change, degraded sensing capacity, and exhaustion of buffers.
42. **Assumes “Survival debt” accumulates roughly linearly but has threshold/step consequences** (works until it doesn’t).
43. **Assumes “Reproduction debt” accumulates as readiness decay** and increases restart cost rather than producing the catastrophic patterns of other debts.
44. **Assumes “Potential” accumulates from Evolution investment not yet converted into Reproduction output**, accumulates in steps, and has a half-life.
45. **Assumes potential conversion is discontinuous and requires “context readiness”** external to the potential-holder (a matching Reproduction opportunity must exist).
46. **Assumes “Overhead” is Survival allocation share** that can ratchet upward across thresholds and persist after stressors diminish.
47. **Assumes a key compound dynamic**: Evolution debt ↔ Survival overhead reinforcing loop, driving degradation sequence.
48. **Assumes potential can mask debt as a buffer** (stored capability absorbs gaps), delaying observability until depletion/departure.
49. **Assumes “metabolic signatures” are recognizable recurring configurations** of contour dominance + accumulation profile (e.g., Starvation, Borrowed Life, Fermentation, Flow, Proliferation, Churn).

### G) Course/Trajectory Assumptions (how systems degrade/recover)

50. **Assumes a staged “system course” progression** (distortion → stress → degradation → breakdown → failure → collapse) is a useful diagnostic structure.
51. **Assumes “distortion” is produced by displacement** and is the entry point; debt begins accumulating before visible output changes.
52. **Assumes “breakdown” is a threshold where prior balance becomes structurally unavailable** (cannot be restored via redistribution alone).
53. **Assumes the model does not claim deterministic outcomes**; it describes structural conditions that make recovery/rebalancing/transformation/reconstitution more or less available.

### H) Cross-Domain Transfer Assumptions (biological, technical, socio-economic, cosmological)

54. **Assumes the structural logic holds across domains** when scarcity + competing demands + non-static behavior are present (explicit generality claim).
55. **Assumes domain differences primarily affect parameters and proxies**, not the core structure (e.g., what counts as “resource,” what signals exist, what buffers look like). *Implied by the model’s domain-general claim plus explicit “resources may be many types.”*
56. **Assumes “unit boundary choice” is analytical**, and valid analysis depends on declaring boundary, time horizon, and environmental conditions (comparability discipline).
57. **Assumes comparisons across domains (or cases) require comparability conditions** (boundary, decomposition level, time horizons, environmental conditions) to avoid false equivalence.

### I) Non-Normativity & Use-Constraint Assumptions (guardrails inside the model)

58. **Assumes the model is descriptive/diagnostic first and not inherently normative**: no contour balance is “inherently superior,” and it does not assume moral/political orientation.
59. **Assumes the model is not a governance/self-organization/complexity theory** and should not be treated as one.
60. **Assumes the model is falsifiable** via structural claims (e.g., if applicability conditions hold but contour tension is not observed; if displacement does not produce distortion).

*   **Medium confidence (implied by stated structure, but not always spelled as a standalone claim):** H55–H57 (domain transfer framed via proxy/parameter variation + comparability requirements).

### Open assumption questions (remaining uncertainty to stabilize before/while testing)

1.  **Boundary declaration:** What boundary definition is being held constant per domain case (what is “inside” vs “environment”)?
2.  **Resource/proxy mapping:** What specific proxy will represent “allocation” in each domain (time, energy, capital, attention, throughput capacity)? (The model allows many resource types but does not prescribe proxies.)
3.  **Signal observability:** What will count as a “signal” and what observation frame (position + temporal resolution) is used, given metabolic frame dependence?
4.  **Mediator identification:** Which mediators are considered operative in each domain case, and at what selectivity/resolution/temporal behavior?
5.  **Mechanism detectability:** What counts as evidence of displacement vs compensation vs feedback in each domain (since manifestation is domain-dependent)?

## ARTIFACT - Assumption Excavator - Assumption-to-Observable Hooks (Multi-Domain Test Scaffold)

**Purpose:** For each “critical” assumption above, state what would have to be observable for it to be accept/reject/testable later—without asserting outcomes.

### 1) Applicability checks (A1–A4)

*   **Assumes limited resources (A1):** Would be testable only if a finite resource budget can be identified at the chosen boundary (some constraint that forces trade-offs).
*   **Assumes competing demands (A2):** Would be testable only if increasing allocation to one contour can be shown to reduce capacity available to at least one other contour (observable opportunity cost).
*   **Assumes non-static behavior over time (A3):** Would be testable only if allocation patterns and/or environmental conditions can be observed to vary across time horizons.

### 2) Contour ontology checks (B6–B7, D23)

*   **Assumes three contours are analytically separable (B6–B7):** Would be testable only if activities/flows can be classified by primary functional intent (even if secondary effects exist), and that classification yields stable distinctions rather than collapsing into one category.
*   **Assumes contour tension exists (D23):** Would be testable only if at least one of the defined trade-offs can be observed under scarcity (e.g., resources shifted to preserve viability reduce resources available for adaptation, etc.).

### 3) Structural element checks (C14–C22)

*   **Assumes Code conditions boundary/gate/signal/receiver (C15–C16):** Would be testable only if there is evidence that the same “pressure” yields different responses in different systems because their operating logic differs (not merely because resources differ).
*   **Assumes Gate/Receiver dependencies for feedback (C19–C22, E34):** Would be testable only if cases exist where feedback signals are generated but do not alter allocation—and the non-effect can be attributed to blocking (Gate), weakness (below Threshold), or illegibility (Receiver).

### 4) Dynamics checks (E26–E37)

*   **Assumes displacement under pressure (E26):** Would be testable only if increased demand or reduced resource envelope correlates with reduced allocation to at least one contour.
*   **Assumes compensation masks distortion (E29–E30):** Would be testable only if observable viability persists despite distortion and there is identifiable buffer expenditure plus signals of that expenditure.
*   **Assumes endogenous phase transition (E35):** Would be testable only if discontinuous allocation shifts occur without a detectable external trigger, with internal accumulation crossing a threshold as the proposed trigger.
*   **Assumes mediator modulation (E36–E37):** Would be testable only if changes in mediators (e.g., governance/trust/power as described) correspond to altered displacement/feedback/compensation patterns, and/or if sustained allocation patterns correlate with mediator reconfiguration (reciprocity).

### 5) Metabolism/accumulation checks (F38–F49)

*   **Assumes accumulation from sustained allocation (F38):** Would be testable only if longitudinal observation shows persistent under-allocation produces measurable capability loss (debt), and sustained Evolution investment produces stored capability awaiting conversion (potential), and sustained stress produces persistent Survival share (overhead).
*   **Assumes debt/potential/overhead interactions (F47–F48):** Would be testable only if masking (potential acting as buffer) or reinforcing loops (debt ↔ overhead) can be observed as coupled dynamics over time.
*   **Assumes metabolic signatures recur (F49):** Would be testable only if multiple cases across domains show recognizable configurations matching the signature definitions (contour dominance + accumulation profile), not just one-off anecdotes.

### 6) Cross-domain generality checks (H54–H57)

*   **Assumes cross-domain generality (H54):** Would be testable only in domains where the applicability conditions (scarcity, competing demands, non-static behavior) can be clearly demonstrated; otherwise a “non-fit” case is underdetermined.
*   **Assumes comparability discipline is required (H56–H57):** Would be testable only if cross-domain conclusions change when boundary/time-horizon/environment are not aligned (the model explicitly flags boundary/temporal/environment mismatch as comparability errors).

### 7) Falsifiability hooks (I60)

*   **Assumes falsifiability via structural claims (I60):** Would be testable only if you can specify a case meeting applicability conditions where (a) contour tension is absent, or (b) displacement does not produce distortion—because those are named falsification routes in the text.
