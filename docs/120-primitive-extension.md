# Tri-Contour Dynamics of Limited Resource Allocation – Primitive Extension Memo

## Purpose

This memo formalizes two additions to the Tri-Contour (TC) foundational model:

- The introduction of **Code** as a fifth primitive — a substrate layer beneath the existing four primitives
- The refinement of **Threshold** into **Receiver** as a richer primitive, with Threshold as a property

These additions emerge from applying the TC model to organizational systems and examining its biological analogues. They extend the model's explanatory range without altering its existing structure.

---

## Prior State of the Primitive Set

The TC model previously operated with four implicit primitives derived from its contour logic:

| Primitive | Role |
|---|---|
| Boundary | Defines inside vs. outside the system |
| Gate | Controls flow across the Boundary |
| Signal | Transmits state-change information |
| Threshold | Determines when a contour responds |

These four were sufficient for describing single-contour dynamics but left two gaps:

- No explicit account of what determines the *meaning* of a Signal — why the same Signal produces different responses in different systems
- No account of the Receiver as a structured actor — Threshold alone described a trigger condition, not the full interpretive apparatus

---

## Code — The Foundational Substrate

### Definition

Code is the substrate that defines the operating logic of a TC system. It precedes and conditions all other primitives:

- Boundary uses Code to distinguish self from non-self
- Gate uses Code to determine what is permitted to cross
- Signal is only legible within a shared Code context
- Receiver is configured by Code to parse specific Signal formats

Code is not the same as *narrative*. Narrative is the expressed story — how Code is read and performed by actors in a system. Code is the underlying sequence being run. The distinction matters: most organizational change efforts operate on narrative while leaving Code untouched, which is why they fail to produce durable change.

### Biological Analogue

In biological systems, Code corresponds to the genome — the master instruction set that:

- Defines what the organism is and how it should respond to stimuli
- Is read and expressed differently depending on context (epigenetics as local narrative)
- Persists across generations (culture as inherited Code in human systems)
- Can be deliberately rewritten — gene editing as the formal mechanism of Code change

The genome is near-fixed narrative in biology. Organizational Code is rewritable in real time — and control over that rewriting is one of the primary stakes of organizational power.

### Organizational Analogue

| Dimension | Biology | Human Organization |
|---|---|---|
| Code substrate | Genome | Shared belief about what the system is and why |
| Writability | Near-fixed; slow mutation | Continuously rewritable; fast with the right tools |
| Who controls it | No single actor | Survival-anchored actors disproportionately |
| Expressed as | Phenotype, behavior | Narrative — values, norms, official story |
| Mutation mechanism | Random or gene-edited | Organic drift or deliberate narrative rewrite |
| Inheritance mechanism | Genetic reproduction | Onboarding, culture transmission, documentation |

### Gene Editing as Formal Code Rewrite Mechanism

Random mutation (organic cultural drift) is the slow path. Deliberate Code rewrite — the organizational equivalent of gene editing — requires four conditions:

| Condition | Biology (CRISPR) | Organization |
|---|---|---|
| Precise targeting | Know which sequence to change | Know which belief or assumption is load-bearing in current Code |
| Delivery vector | Machinery to enter the cell | Trusted actor, crisis, or external event that carries new Code inside the Survival boundary |
| Immune suppression | Prevent rejection before edit takes | Sufficient legitimacy or disruption to weaken the system's rejection response |
| Expression conditions | Edit must be read in active context | The system must actually be running the Code for the edit to propagate |

Failure at any condition produces a predictable outcome:

- **No targeting:** change theater — activity without Code contact
- **No vector:** signal never crosses the Survival boundary
- **No immune suppression:** actor expelled; edit rejected
- **No expression:** edit lands but never activates — the system says the right things and does nothing differently

### The Drunk Organism Condition

> *"The organism is drunk enough that the liver gets a vote."*

A named state in which systemic stress is sufficient to dissolve normal boundary enforcement. Under the drunk organism condition:

- Survival-contour boundary gates are temporarily open or overwhelmed
- Evolution-anchored actors gain unusual access — their signals can propagate without normal immune rejection
- Code rewrite becomes possible without satisfying all four gene editing conditions
- The window is brief — the immune response reconstitutes, often more aggressively than before

Organizational triggers: crisis, bankruptcy, acquisition, rapid leadership replacement, existential competitive threat. Not politics. Not narrative. Actual system stress that dissolves the existing boundary map.

## Receiver — Refined Primitive

### Rationale for Refinement

Threshold as originally stated described a trigger condition: the minimum signal strength required for a contour to respond. This was necessary but insufficient. It did not account for:

- Whether a Signal is parseable at all by the receiving actor
- Which Signal formats the actor is primed to detect
- The contour orientation that shapes what the actor is looking for

Receiver replaces Threshold as a primitive. Threshold becomes a property of Receiver.

### Definition

A Receiver is an actor or subsystem configured by Code to detect, parse, and respond to specific Signal formats. A Receiver has three properties:

| Property | Definition | Biological analogue |
|---|---|---|
| Threshold | Minimum signal strength required to trigger a response | Action potential; fever trigger |
| Legibility range | The set of Signal formats the Receiver can parse, as defined by Code | Receptor protein shape — molecular binding specificity |
| Contour orientation | Which contours the Receiver is primed to monitor for signals | Immune cell vs. neural cell vs. endocrine receptor — each tuned to different signal types |

### Legibility as Derived Property

Legibility is not a primitive. It is a compatibility condition between Signal and Receiver — specifically, whether the Signal format falls within the Receiver's legibility range as defined by shared Code.

A Signal that falls outside a Receiver's legibility range produces no response — not because a Gate blocked it, not because it failed to reach Threshold, but because the Receiver has no category for it. The Signal lands and is inert.

This explains a common organizational failure: an Evolution-anchored actor generates a well-formed signal, no gate blocks it, it reaches decision-makers — and nothing happens. The Receivers are not configured by their Code to parse signals of that type.

## Revised Primitive Set

| Primitive | Layer | Role | Biological instance |
|---|---|---|---|
| Code | Substrate — precedes all others | Defines the operating logic; conditions all other primitives | Genome |
| Boundary | Structure | Defines inside vs. outside using Code as reference | Cell membrane; immune self/non-self recognition |
| Gate | Structure | Controls flow across Boundary | Ion channels; blood-brain barrier |
| Signal | Dynamics | Transmits state-change information within a Code context | Hormones; neural firing; cytokines |
| Receiver | Dynamics | Interprets Signal according to Code; fires at Threshold | Receptor proteins; immune cells; neural circuits |

Code is foundational — it is not at the same layer as the other four primitives. It is the substrate that determines how each of the other four operates in a given system instance.

## Implications for TC Architecture

### Layer 1 — Model

The primitive set requires updating to reflect five primitives with Code at substrate level. The existing Boundary/Gate/Signal structure is unchanged; Threshold is absorbed into Receiver; Code is added as the foundational layer.

### Layer 3 — Dynamics

Gene editing as a formal Code rewrite mechanism should be documented as a dynamics pattern — distinct from Signal propagation. It operates on Code directly, not through normal contour channels.

The drunk organism condition should be named and defined as a system state that temporarily suspends normal gate enforcement, enabling Code access that would otherwise be blocked.

### Layer 4 — Failure

The biological analogy surfaces three failure modes that map cleanly to organizational pathology:

| Failure mode | Biology | Organization |
|---|---|---|
| Autoimmune | Immune system attacks self — Survival actor misidentifies inside as threat | Management optimizing against its own workforce; gatekeeping that expels value-generating actors |
| Cancer | Reproduction escaped Survival regulation — replication without boundary recognition | A function that scales without constraint; pure Reproduction with no Evolution input and no Survival accountability |
| Membrane impermeability | Membrane blocks signal in both directions — isolation failure | Middle management blocking signal upward and downward; information stops propagating |

### Layer 5 — Application

The AI/organization observation from which this memo emerged is a candidate worked example for Layer 5. It illustrates:

- Evolution-anchored actors (employees with AI) gaining Signal articulation without crossing into Survival contour
- Code rewrite conditions becoming cheaper on the targeting dimension while remaining blocked on vector and immune suppression
- Survival-anchored actors tightening entry filters in response — a Compensation mechanism absorbing Evolution pressure rather than transmitting it

## Open Questions for Review

The following are flagged for review before these additions are promoted to canonical model status:

- Is Code truly irreducible, or can it be derived from the interaction of the other four primitives? Test: can you describe a system without invoking Code explicitly?
- Does the Signal/Receiver pairing need formal notation — a transaction model that defines what a complete Signal-Receiver interaction looks like?
- Are the three biological failure modes (autoimmune, cancer, membrane impermeability) exhaustive at the primitive level, or are there others?
- The narrative layer in human systems is rewritable in real time — does this require a separate mechanism, or is it fully captured by the gene editing conditions?
- Where does the drunk organism condition sit in the six-layer architecture — Dynamics (Layer 3) or Failure (Layer 4)?
