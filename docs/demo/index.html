TC Dynamics — Scenario 01

Starvation

A balanced system absorbs an environmental shock. Evolution is displaced first. Debt accumulates, overhead rises — the reinforcing loop initiates and drives the system toward degradation.
TC Dynamics — Scenario 02

Borrowed Life

The system is Survival-dominant — but it doesn't look that way. Accumulated potential in experienced actors absorbs the growing E-debt, masking decline. Output appears stable until the buffer exhausts. Then the debt becomes visible all at once.
TC Dynamics — Scenario 03

Proliferation

The system scales aggressively. Growth metrics climb, reinforcing feedback locks in Reproduction dominance. Survival and Evolution debt accumulate in parallel — invisibly. Both thresholds cross simultaneously, producing rapid degradation from apparent health.
TC Dynamics — Scenario 04

Churn

The system invests heavily in Evolution — but no investment persists long enough to mature. Each initiative is replaced before it converts to output. Potential never accumulates. There is no debt crisis, no collapse. Only permanent opportunity cost: the growing gap between what sustained investment could have produced and what constant reset actually yields.
TC Dynamics — Scenario 05

Successful Recovery

Displacement begins. E-debt starts accumulating. But balancing feedback is generated, crosses the Gate, reaches a Receiver above threshold — and is parsed in time. Reallocation corrects the distortion before debt compounds. The recovery window stays open. The system returns to balance.
TC Dynamics — Scenario 06

Missed Recovery Window

Identical initial conditions to Scenario 05. Same displacement, same signal generated. But compensation masks the signal long enough that detection arrives late. By the time reallocation is attempted, the reinforcing loop is already active. The window has closed. The same action that would have worked earlier now cannot.
