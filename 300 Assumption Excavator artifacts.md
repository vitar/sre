# ARTIFACTS

## ARTIFACT - Assumption Excavator - Critical Assumptions Sorted

### Foundational critical assumptions

These are assumptions the model presents as part of its core geometry or structural logic.

| ID | Assumption                                                                                                                                                | Why it is foundational                                             |
| -- | --------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------ |
| F1 | Assumes that viable dynamic systems under meaningful resource limitation allocate across **three** core contours: Survival, Reproduction, and Evolution.  | This is the model’s primary structural claim.                      |
| F2 | Assumes that all three contours remain analytically relevant even when one is hidden, displaced, or blocked.                                              | This protects the tri-contour frame from apparent counterexamples. |
| F3 | Assumes that allocation to one contour constrains allocation to the others over the relevant horizon.                                                     | This creates the model’s trade-off logic.                          |
| F4 | Assumes that the system’s meaningful state can be described through contour distribution.                                                                 | This makes “state estimation” possible at all.                     |
| F5 | Assumes that complexity and nesting do not require a different core geometry; they add interacting local allocations instead.                             | This preserves the model’s universality across scale.              |
| F6 | Assumes that major cross-cutting factors are better treated as mediators than as additional core contours.                                                | This protects the model from contour proliferation.                |

### Empirical critical assumptions

These are assumptions about recurring real-world regularities the model expects to observe.

| ID | Assumption                                                                                                                              | Why it is empirical                                                        |
| -- | --------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| E1 | Assumes that prolonged contour dominance tends to produce characteristic distortions.                                                   | This is a real-world pattern claim, not just a definitional one.           |
| E2 | Assumes that simultaneous sustainable maximization of all three contours does not occur under meaningful scarcity.                      | This is a strong regularity claim about constrained systems.               |
| E3 | Assumes that deep system behavior can often be understood by examining contour allocation, mediators, and feedback.                     | This is the model’s broad explanatory claim.                               |
| E4 | Assumes that viable continuity is the right anchor for diagnosing failure, including cases of formal survival with functional failure.  | This is an empirical claim about how failure actually shows up in systems. |

### Operational critical assumptions

These are assumptions required to apply the model reliably rather than loosely.

| ID | Assumption                                                                                                                                          | Why it is operational                                                                    |
| -- | --------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| O1 | Assumes that primary functional intent can usually be identified well enough to classify objects.                                                   | Without this, contour assignment becomes unstable.                                       |
| O2 | Assumes that mediator-first handling is sufficient for cross-cutting objects and avoids forced misclassification.                                   | This is how the model handles governance, trust, power, and similar factors in practice. |
| O3 | Assumes that proxy-supported inference can credibly estimate contour posture when direct allocation is hidden.                                      | This supports serious use in opaque real systems.                                        |
| O4 | Assumes that boundary and time-horizon declaration can discipline interpretation rather than merely relativize it.                                  | This keeps variation in diagnosis from becoming arbitrary.                               |
| O5 | Assumes that comparability can be established conditionally through aligned artifacts, boundaries, horizons, proxy logic, and mediator conditions.  | This enables comparison without false equivalence.                                       |
| O6 | Assumes that better discipline and artifact traceability materially improve reliability.                                                            | This is the model’s safeguard against interpretive drift.                                |

## ARTIFACT - Assumption Excavator - Fatal If False Versus Survivable If False

| F2 | All viable dynamic systems under relevant conditions allocate across all three contours in some form.  | Failure creates true contour-free or missing-contour systems inside the model’s intended scope. |
| F3 | Allocation to one contour constrains the others over the relevant horizon.                             | Failure removes the model’s core tension and trade-off logic.                                   |
| F4 | System state is meaningfully expressible through contour distribution.                                 | Failure removes the model’s state-description engine.                                           |
| F6 | Cross-cutting factors can be treated as mediators without becoming extra core contours.                | Failure means the tri-contour core is structurally incomplete.                                  |

## ARTIFACT - Assumption Excavator - Fatal Assumptions Ranked From First to Test to Last to Test

This rank is based on **diagnostic leverage**: earlier tests target assumptions that would invalidate the most downstream structure with the least interpretive overhead.

| Test order | Fatal assumption                         | Why test here                                                                                                                                                                       |
| ---------- | ---------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1          | **F1 — Three-contour sufficiency**       | This is the shortest path to falsifying the model’s core architecture. If a fourth irreducible contour is needed inside scope, much else becomes secondary.                         |
| 2          | **F6 — Mediator-not-contour treatment**  | This is the most likely structural breach mechanism, since the model explicitly pushes governance, power, legitimacy, and meaning into mediator status.                             |
| 3          | **F2 — Universal tri-contour relevance** | After testing sufficiency, test universality: are there viable in-scope systems missing one contour in a non-displaceable way?                                                      |
| 4          | **F3 — Constraint through allocation**   | If trade-offs do not reliably hold, the model’s core tension logic weakens sharply.                                                                                                 |
| 5          | **F4 — State by distribution**           | This should be tested after the structural ontology, because it depends on the contours being real and sufficient before asking whether distribution actually tracks system state.  |

### Reading note

The first two tests target whether the model has the **right ontology**.
The next two test whether that ontology is **universally present and tension-generating**.
The last tests whether the ontology is **actually explanatory and diagnostically usable**.
